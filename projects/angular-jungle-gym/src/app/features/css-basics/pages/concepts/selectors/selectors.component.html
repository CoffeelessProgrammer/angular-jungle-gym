Core Selectors

  • Classes (.)
    • CSS - Start with a period, e.g. .blog-post-heading
    • HTML - Specified in class attribute, e.g. class="blog-post-heading"
    • Apply multiple classes to a single element via space separation
    • Each class can applied to multiple elements
  • IDs (#)
    • CSS - Start with a hash symbol, e.g. #blog-post-heading
    • HTML - Specified in id attribute, e.g. id="blog-post-heading"
    • Applied to single element with id
  • Attributes ([...])
    • CSS - Enclose HTML attribute with square brackets, e.g. [disabled]
    • HTML - Attribute on HTML element, e.g. <button disabled>Next</button>
  • Elemental
    • Selects all instances of an HTML element/tag, e.g. h1, p, div
    • HTML Elements Reference | W3Schools
  • Universal (*) - Selects all elements; useful for CSS resets

Combinators

  • Descendant ( )
    • Second element is a descendant (child) of first element, e.g.
    • nav li { ... } - selects all list elements which have a <nav> parent in their hierarchy
  • Child (>)
    • Second element is a direct child of first element, e.g.
    • nav > a { ... } - selects all anchor tags which have a <nav> parent
  • General Sibling (~)
    • Second element comes after first element and shares the same parent, e.g.
    • h4 + p { ... } - selects all paragraphs which follow an h4 tag on the same hierachy level
  • Adjacent Sibling (+)
    • Second element comes immediately after first element (same parent), e.g.
    • h3 + p { ... } - selects paragraphs which directly follow an h3 tag

Selector Precedence (Specificity)

  1. Inline styles
  2. #id
  3. .class, :pseudo-class, [attribute]
  4. <tag>, ::pseudo-element

Elements also inherit certain properties from parent elements (low specificity), e.g. text elements inherit font-family from <body>. Inheritance is easily overridden.
